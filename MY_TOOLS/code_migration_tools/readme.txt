假设有两个文件夹mmd_rs和mmd，mmd中的是原始代码，
mmd_rs是mmd的修改版本，可以更改的规则为：
1. 对mmd的任何修改体现为添加新文件，原始文件不能修改，当出现需要修改原始mmd的代码时，体现为
另起一个文件，并另外去一个函数名。
2. __init__中需要添加新的函数时，在原始__init__的下方#################的分割

compare_and_split：
比较当前基于mmd的项目与原始mmd的不同，并将只在当前项目中出现的文件挑选出来(split)。
同时，对于__init__文件，也挑选出来。

merge_compare：
将compare_and_split的split结果复制到新的mmd项目中，
并且将__init__文件中##############以下的部分也添加。
之后重新编译、配置环境等即可。
